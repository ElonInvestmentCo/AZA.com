# UI Components

## Buttons

Buttons are primary action triggers across PAYVORA, designed for clarity and financial confidence.

- Variants: Primary, Secondary, Ghost, Destructive
- Minimum height: 44px for accessibility
- States: default, hover, active, disabled, loading
- Used for payments, transfers, confirmations

---

## Inputs

Inputs capture structured financial and user data.

- Floating labels for clarity
- Inline validation and error messaging
- Auto-formatting for currency, phone numbers, IDs
- Secure input modes for sensitive data (PIN, password)

---

## Dropdowns

Dropdowns provide controlled selection of predefined options.

- Search-enabled for large datasets
- Used for currency, country, providers
- Keyboard and touch accessible
- Clear selected state display

---

## Cards

Cards are the primary financial information containers.

- Used for wallet balances, crypto assets, transactions
- Soft elevation and rounded corners
- Clear hierarchy: title → value → metadata
- Tap-friendly for mobile interactions

---

## Tables

Tables display structured financial datasets.

- Used in dashboards and admin views
- Sortable columns (date, amount, status)
- Sticky headers for long datasets
- Responsive horizontal scrolling

---

## Charts

Charts visualize financial trends and analytics.

- Line charts: balances, performance trends
- Bar charts: spending categories, comparisons
- Pie charts: portfolio allocation
- Real-time update capability

---

## Tabs

Tabs organize content into logical sections.

- Maximum 5 tabs per view
- Active tab highlighted with primary color
- Smooth transitions between sections
- Swipe support on mobile

---

## Modals

Modals handle critical user actions and confirmations.

- Used for transfers, confirmations, warnings
- Focused single-task interaction
- Background blur for context retention
- Clear confirm/cancel actions

---

## Toasts

Toasts provide temporary system feedback.

- Success, error, and info variants
- Auto-dismiss behavior
- Non-blocking notifications
- Used for transaction confirmations

---

## Alerts

Alerts communicate persistent or critical system states.

- Inline or banner style
- Severity levels: info, warning, error
- Requires acknowledgment for critical cases
- Used for compliance or security messages

---

## Badges

Badges indicate status or metadata.

- Transaction status indicators
- Color-coded semantic meaning
- Compact UI elements
- Used on cards and lists

---

## Chips

Chips are compact interactive filters.

- Multi-select filtering support
- Removable tags
- Used for categories and transaction filtering
- Rounded pill design

---

## Navigation Bar

Primary navigation structure for desktop/tablet.

- Wallet always accessible
- Core sections: Home, Pay, Trade, Cards, Profile
- Persistent layout
- Active state indicators

---

## Bottom Navigation

Primary mobile navigation system.

- Fixed bottom placement
- Maximum 5 tabs
- Icon + label structure
- Thumb-friendly interaction

---

## Floating Action Button

Quick access primary action trigger.

- Used for Send / Pay / Add Money
- Positioned bottom-right on mobile
- Expands into quick action menu
- Visually distinct from UI elements

---

## Wallet Card

Core financial summary component.

- Displays total balance prominently
- Multi-currency support
- Quick actions: send, receive, fund
- Real-time updates

---

## Transaction Card

Represents individual financial activity.

- Amount, type, status
- Timestamp and counterparty
- Status badge integration
- Tap for detailed view

---

## Crypto Card

Displays digital asset holdings.

- Asset name and symbol
- Live price updates
- 24h change indicator
- Portfolio allocation percentage

---

## Gift Card Card

Represents digital gift card assets.

- Brand, value, and status
- Redeem or transfer actions
- Secure code handling
- Marketplace visibility status
