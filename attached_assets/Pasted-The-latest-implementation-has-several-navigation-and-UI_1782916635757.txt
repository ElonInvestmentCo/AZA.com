The latest implementation has several navigation and UI issues that need to be fixed before making any additional changes.

## 1. Remove duplicated pages

There should only be **one implementation** of every screen.

Currently, multiple pages appear to be duplicated or recreated instead of navigating to the existing screens.

Examples include:

* Dashboard/Home
* More
* Settings
* Bills
* Card screens

Do **not** create cloned versions.

Instead:

* Reuse the existing routes.
* Reuse existing components.
* Reuse existing navigation.
* Remove duplicate screens and duplicate route registrations.
* Every button should navigate to the existing page only once.

---

## 2. Fix navigation

Several buttons either:

* do nothing,
* open the wrong page,
* open duplicate pages,
* or are not connected.

Every interactive element must work.

Verify every navigation path:

* Fund Wallet
* Send
* Withdraw
* Gift Cards
* Airtime
* Electricity
* Cable TV
* Rates
* Transactions
* Betting
* Funding
* More
* Settings
* Notifications
* Card section
* Recent Transactions
* View All
* Bottom Tab Navigation

Every button should navigate exactly once to its intended destination.

---

## 3. Fix non-responsive buttons

Some buttons require multiple taps or do not respond.

Inspect every:

* TouchableOpacity
* Pressable
* Button
* Card
* Icon button

Ensure:

* onPress is connected.
* touch area is large enough.
* no invisible View is blocking touches.
* pointerEvents are correct.
* zIndex does not prevent interaction.
* scrolling does not interfere with taps.

Every button should respond instantly.
