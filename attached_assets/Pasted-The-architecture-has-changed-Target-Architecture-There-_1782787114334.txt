The architecture has changed.

## Target Architecture

There should be only **two frontends** in this repository:

1. **PayVora Website**

   * Location: `artifacts/website`
   * Framework: Next.js (App Router)
   * TypeScript
   * Tailwind CSS
   * Framer Motion
   * Responsive
   * SEO optimized
   * This becomes the public website at **payvora.org**.

2. **PayVora Mobile**

   * Location: `artifacts/mobile`
   * Expo
   * React Native
   * Expo Router
   * Android
   * iOS
   * Expo Go only

The mobile app is **not** intended to be the public website anymore.

## Expo Web

We are **retiring the Expo Web application**.

Do **not** spend development effort maintaining or improving Expo Web.

Do **not** delete any files unless they are confirmed to be unused.

Leave existing web-related files intact if removing them could affect native builds.

The only required change is:

Replace the Replit workflow named:

```
AZA Mobile Web
```

with

```
PayVora Website
```

that starts the Next.js website.

## Important

Do not create another repository.

Do not move the mobile app.

Do not break Expo Go.

Keep everything inside the existing monorepo.

Before changing any existing configuration, perform a dependency audit and preserve anything required for Android, iOS, or Expo Go.

After implementation, verify that:

* the API server still runs,
* the mobile app still runs in Expo Go,
* the new Next.js website runs successfully,
* all workflows start correctly,
* no mobile functionality has regressed.
